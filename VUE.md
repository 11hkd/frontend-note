# Vue Router

Vue Router 是 Vue 的一个插件库，用于构建单页面应用。它的本质是建立 URL 路径和页面之间的映射关系。在 Vue Router 的单页面应用中，页面路径的改变就是组件的切换。路由主要负责将浏览器请求映射到特定的组件代码中，通过解析 URL 实现页面切换和跳转。

## 什么是路由懒加载？

路由懒加载是指不用一开始加载所有页面，而是用到哪里就加载哪里，避免不必要的资源加载。没有路由懒加载，Webpack 打包后的文件会异常大，加载内容过多，速度过慢，会导致用户进入页面白屏，影响体验。

## 如何实现 Vue 路由跳转

1. **声明式导航**
2. **编程式导航**：在函数里面调用路由地址，通常在逻辑跳转时使用。

## Vue 2 和 Vue 3 的区别

1. **根节点**：Vue 2 只支持一个根节点，Vue 3 支持多个根节点。
2. **API**：Vue 2 使用选项式 API，Vue 3 使用组合式 API。选项式 API 功能太分散，代码难以复用，不适合大项目。组合式 API 打包时，自动将没有用到的功能删除。每个功能的 methods 和 data 都可以封装在一个函数中。
3. **生命周期**：生命周期发生了改变，改了几个名字，并加了两个钩子函数。
4. **核心原理**：Vue 2 是通过 `Object.defineProperty` 数据劫持结合订阅发布模式实现双向数据绑定的，Vue 3 是通过 Proxy 数据代理来实现的。
5. **父子传参**：setup 函数特性，有 return。

## 组件开发的好处

- 提升整个项目的开发效率，避免重复的工作量。
- 提高复用性。
- 把页面抽象成多个相对独立的模块，使代码逻辑清晰，方便项目的后期维护。
- 便于协同开发，不用把大量精力放在内部的实现上，实现模块化开发。

### 传统流程

1. 创建 `Xxx.vue` 文件，对组件进行封装。
2. 通过 `import` 的方式将组件导入：`import Xxx from '...'`
3. 对组件进行注册：`components: { Xxx }`
4. 使用组件：`<Xxx/>`

## VueX 状态管理实现原理

把多个组件需要共享的变量全部储存在一个对象里面，然后将这个对象放在顶层的 Vue 实例中，让其他组件可以使用，并且让这个对象里面的属性做到响应式。

## Vue 的特性

1. **数据驱动视图**：Vue 会监听数据变化，当页面数据发生变化时，Vue 会自动重新渲染页面结构。
2. **双向数据绑定**：当数据发生变化时，视图也会发生变化；当视图发生变化时，数据也会跟着同步变化。
3. **指令**：包括内置指令和自定义指令。内置指令是 Vue 自带的指令，自定义指令是用户自己定义的指令。指令名称以 `v-` 开头，作用于 HTML 中的元素。
4. **插件**：Vue 支持插件，通过加载插件可以实现更多功能。

## MVVM 的组成部分及基本工作原理

MVVM 主要包含 Model（数据模型）、View（视图）和 ViewModel（视图模型）。其中，Model 是指数据部分，负责业务数据的处理；View 是指视图部分，即用户界面，负责视图处理；ViewModel 用于连接视图与数据模型，负责监听 Model 或者 View 的改变。
